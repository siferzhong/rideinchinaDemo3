# 管理员账号设置说明

## 权限逻辑说明

- **与 WordPress 会员的关系**：本 App 的角色（管理员 / 领队 / 普通用户）**独立于** WordPress 自带的“订阅者 / 作者 / 编辑 / 管理员”等角色。
- **管理员判定**：满足以下**任一**即视为 App 管理员：
  1. WordPress 用户元数据 `rideinchina_role` = `admin`
  2. 该用户在 WordPress 中是 **Administrator（管理员）**
- 也就是说：**用 WordPress 后台的“管理员”账号登录 App，自动拥有 App 管理员权限**；也可以给任意 WordPress 用户单独加 `rideinchina_role`，不一定要用 WP 管理员。

---

## 如何给自己设置成管理员

### 方式一：使用 WordPress 管理员账号（推荐）

1. 在 WordPress 后台 **用户 → 用户** 里，确保你的账号角色是 **管理员（Administrator）**。
2. 用该账号在 App 里 **登录**（用户名 + 密码）。
3. 登录后 App 会识别为管理员，底部会出现 **Admin** 标签，可管理证件、用户、群目的地、群消息等。

无需改任何自定义字段，WP 管理员 = App 管理员。

---

### 方式二：给任意用户加“App 管理员”角色

若你希望某个账号**不是** WordPress 管理员，但在 App 里当管理员：

1. 登录 **WordPress 后台**。
2. 进入 **用户 → 所有用户**，点击要设置的用户 **编辑**。
3. 在用户编辑页面向下找到 **自定义字段（Custom Fields）**。
   - 若没有看到，点击“显示更多”或“选项”里的“自定义栏目”。
4. 新增一条自定义字段：
   - **名称（Name）**：`rideinchina_role`
   - **值（Value）**：`admin`
5. 点击 **添加自定义栏目**，然后 **更新用户**。

之后该用户用 App 登录即可拥有管理员权限。

**领队**同理：自定义字段 `rideinchina_role` = `leader`。  
**普通用户**：不设该字段或设为 `user` 即可。

---

## 在 App 里管理员能做什么

- **Documents**：查看所有用户上传的证件/图片，审核通过或拒绝。
- **Users**：查看用户列表，将用户设为 admin / leader / user。
- **Destination**：设置或清除群组骑行目的地。
- **Messages**：发送群消息；管理员/领队消息在群聊中会高亮显示。

---

## 查看用户上传的图片/证件

- 在 App 底部进入 **Admin** 标签 → **Documents**。
- 列表按用户分组，每个用户下有证件类型、文件名、状态。
- 点击证件行的 **查看** 可预览用户上传的图片；支持通过/拒绝审核。

管理员**不需要**在 WordPress 后台单独找图片，所有操作在 App 的 Admin → Documents 中完成。
